## v37.3 (2026-04-20) — Minor · hedge 규정 정밀화 (9종 허위정밀 제거)


**근거:** 사용자 지적(2026-04-20) "9종 필요한가 재점검" → 4중 결함 진단:
1. **중복 과잉** — 9종 중 "수 있" 줄기 변형이 4개(1,2,8,9). 실질 고유 패턴 4~5개뿐인데 9로 부풀려짐
2. **치명적 누락** — 고빈도 hedge 미포함: `~로 보인다`(학술 최빈), `~듯하다`, `~ㄹ 것이다`(전망), `대체로·다소·아마`(부사형)
3. **임의성** — "9"는 이론 근거 0. v35.8 autoloop가 우연히 뽑은 개수. 8·10·5도 가능했음
4. **오탐 위험** — 단순 줄기 매칭으로 능력표현("답할 수 있다")과 가능성표현("정답일 수 있다") 구분 불가

**상위 레버 재발견:** 확신도 병기가 본 가드의 상위 원리. 모든 주장에 (높90·보70·낮50·모30) 병기되면 hedge 어휘는 논리적 불필요. hedge 금지는 확신도 의무의 파생 규칙.

**3안 제시 → C안 채택:**
- A (원칙 중심·극단압축) — 재현율↑ but 트리거 구체성↓
- B (기능 4버킷) — 구조적 but 3줄로 확장
- **C (핵심 5 + 원칙 1줄)** ⭐ — 줄수 동일, 포괄성·구체성 양립

**결정 근거:** 줄수 비용 0, 허위 정밀("9종") 제거, 중복 제거(수 있 4→1), 포괄어("어미 전반") 재현율↑, "확신도 병기" 명시로 상위 레버 결합.

**임팩트:** 본체 1줄→1줄 (동일). INVARIANT 7축 BYPASS 없음(의미 동일, 포괄성 확장). 체크리스트 v4.2→v4.3 동기(항목 #7 문구 갱신). 파일명 v37.2.md→v37.3.md 리네임.

**검증:**
